Gemini CLI & Bluetooth - Q&A Summary

Question 1: How can I ensure that Gemini CLI retains the chat context between terminal sessions?

To retain chat context in Gemini CLI between terminal sessions, use named sessions. This allows you to resume your work seamlessly.

To start a named session:
Start a new named session
gemini --session my-project-session

This saves your conversation under a session name. You can close the terminal and return to the same session later.

To resume a saved session:
Resume an existing session
gemini --session my-project-session
To view existing sessions:
List all sessions
gemini sessions list
Optional optimization:

Use the '/compress' command before exiting to summarize and reduce context size, improving performance when you resume later.

For convenience, create an alias:
Add to your shell configuration:
alias gemdev='gemini --session dev-chat'
Avoid starting Gemini CLI without the '--session' flag, as this will initiate a new session each time, discarding prior context.
Question 2: Could you please explain the '/compress' command in Gemini CLI in more detail?

The '/compress' command in Gemini CLI is used to manage context efficiently in long-running conversations. It works by replacing the full conversation history with a high-level summary, allowing the system to retain the key points while minimizing token usage.

This is especially important because large language models have a limited context window (measured in tokens). As a conversation grows, it consumes more tokens, which can lead to performance degradation or forgotten earlier parts of the discussion.

When you run '/compress', Gemini generates a concise summary of the discussion and discards the full message history. This summary becomes the new base context.

Best practices for using compression:

  • Use '/compress' when a topic or task is complete, or before switching to a new subject.
  • Ensure that all key points you wish to retain are clearly mentioned before compressing.
  • Avoid using it in the middle of a highly technical or ongoing thread unless necessary for performance.

This feature helps keep sessions efficient and focused while still preserving continuity through summarization.
